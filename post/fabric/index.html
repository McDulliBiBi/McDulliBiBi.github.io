应用服务器越来越多，每次发布都需要在不同的机器上敲同样的命令，太痛苦了。为了解决这个问题，决定尝试用用fabric。

fabric是一个python库，采用ssh的原理实现在一台机器上部署多台机器的应用，简化发布流程
中文官方文档：https://fabric-chs.readthedocs.io/zh_CN/chs/tutorial.html

安装说明

安装python

  1. 过程： ？
  2. 查看python版本：python -V

安装Yum

  1. 查看是否安装yum rpm -qa | grep yum

  2. 安装yum

    wget http://yum.baseurl.org/download/3.2/yum-3.2.28.tar.gz
tar xvf yum-3.2.28.tar.gz
cd yum-3.2.28
sudo touch /etc/yum.conf
sudo yummain.py install yum

安装python-devel

  1. 查看是否安装python-devel rpm -qa | grep python-devel
  2. 安装python-devel sudo yum -y install python-devel

安装pip

  1. 联网安装： 下载py文件：https://bootstrap.pypa.io/get-pip.py curl https://bootstrap.pypa.io/get-pip.py > get-pip.py python get-pip.py / sudo python get-pip.py

  2. 离线安装：（因为系统组把服务器的网络访问权限关了摊手）
    2.1 下载setuptools及pip安装包到本地，然后上传到服务器。

    • setuptools:
      https://pypi.python.org/packages/45/29/8814bf414e7cd1031e1a3c8a4169218376e284ea2553cc0822a6ea1c2d78/setuptools-36.6.0.zip#md5=74663b15117d9a2cc5295d76011e6fd1
    • pip:
      https://pypi.python.org/packages/11/b6/abcb525026a4be042b486df43905d6893fb04f05aac21c32c638e939e447/pip-9.0.1.tar.gz#md5=35f01da33009719497f01a4ba69d63c9
      2.2 在服务器上解压缩两个安装包
      2.3 先装setuptools，然后安装pip。分别在两个路径下执行sudo python setup.py install

  3. 用pip安装python库： pip install fabric或者pip install fabric==1.14.0指定库的版本号

  4. 用pip卸载python库： pip uninstall fabric

  5. 查看机器当前安装了哪些包： sudo pip freeze

安装fabric

  1. pip install fabric==1.14.0
  2. 查看安装的fabric版本： sudo pip show fabric 或者 sudo pip freeze

用法：

  1. 简单用法，在本地执行命令：建一个名称为fabfile.py文件

    def hello():
print "hello mcdull";

    然后在当前路径下敲命令 fab hello，指定方法名，会打印”hello mcdull” 注：脚本可以建成其他名字，但是执行是需要加 -f 指定文件名。例如：新建了一个print.py脚本，则执行fab hello -f print.py

  2. 在其他机器上远程执行命令：

    # -*- coding: utf-8 -*-
from fabric.api import *

env.hosts = ['1.1.1.0' , '1.1.1.1']
env.user = 'user'
#私钥路径
env.key_filename = "~/.ssh/id_rsa"

@task(default=True)
def default():
run("mkdir /Data/tmp/mcdull3")

然后敲命令fab会默认执行default()方法内的命令

  1. 如果想在不同的机器上执行不同的命令，可以用装饰器@roles

其他

  1. 远程执行时设置ssh免密登录。因为系统组不给appadmin账号的密码，所以用ssh生成密钥的方式免密登录。步骤如下： 1.1 生成公钥，私钥对 ssh-keygen 1.2 进入生成密钥文件的目录 cd ~/.ssh 应该至少有三个文件：私钥 id_rsa，公钥id_rsa.pub，以及 known_hosts。 1.3 新建一个authorized_keys文件，修改文件权限为600，并将其他机器生成的公钥内容输入至authorized_keys touch authorized_keys chmod 600 authorized_keys cat id_rsa.pub >> authorized_keys

  2. fabric的远程执行命令和ssh是一样的原理：
    ssh username@1.1.1.0

  3. 是否几台机器上都要安装fabric？

遇到的异常

  1. >ImportError: No module named api 方法1. 用sudo fab 方法2. 使用1.*版本的fabric,不能用2.*，3.*的，因为高版本的不支持from fabric.api import *这种用法

  2. No handlers could be found for logger “paramiko.transport” paramiko是一个打日志的库？在python脚本里加上

    import logging
import paramiko

paramiko.util.log_to_file("/Data/tmp/mcdull/loginfo.log", level = "WARN")

然后看打印日志报的异常： >from_buffer() cannot return the address of the raw string within a str or unicode or bytearray object
网上找原因说可能是需要安装netmiko：sudo pip install netmiko -U。装好后成功解决
